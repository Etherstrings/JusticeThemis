# 隔夜事件证据与判断设计

- 日期：2026-04-06
- 状态：Approved
- 适用仓库：`daily_stock_analysis`
- 设计主题：在隔夜事件页与晨报页中展示更详细的新闻证据内容，并补一条对应判断

## 1. 设计结论

当前 JusticeThemis 已经能展示：

- 事件标题
- 来源链接
- A 股映射

但用户需要的不只是“有链接”，而是：

- 能直接看到抓到的新闻是什么
- 不用点开原文就能读到一层摘要
- 每条事件旁边有一句更直接的判断

本轮设计采用：

- `证据层`：展示 1-3 条事件新闻证据卡
- `判断层`：展示 1 条一句话判断

判断层允许接入模型，但不是强依赖。

## 2. 范围

### 2.1 要做

- 后端事件详情返回结构化 `evidence_items`
- 后端事件详情返回 `judgment_summary`
- 前端事件详情页展示证据卡与一句判断
- 晨报主页面在当前事件区也展示同样的证据卡与一句判断

### 2.2 不做

- 不展示全文
- 不做多段长篇研报
- 不做链式推理过程曝光
- 不在本轮做批量异步缓存系统

## 3. 数据结构

后端事件详情新增：

- `source_links`
- `evidence_items`
  - `headline`
  - `source_name`
  - `url`
  - `summary`
  - `source_type`
- `judgment_summary`
- `judgment_mode`
  - `model`
  - `heuristic`

## 4. 证据生成策略

优先级：

1. 使用当前事件挂接的一手 `source_links`
2. 根据链接和 source registry 推导来源名
3. 根据 URL path、事件 summary、事件 core fact 生成可读标题与摘要

本轮不要求真实抓取网页全文；只要求比“裸 URL”明显更可读。

## 5. 判断生成策略

### 5.1 推荐方案

先做“模型优先，规则兜底”：

- 若配置了 Gemini / Anthropic / OpenAI 任一可用模型
  - 用事件信息 + 证据摘要生成一句判断
- 若没有模型可用
  - 用现有 event decision / A 股映射生成一句判断

### 5.2 输出要求

判断必须是：

- 一句话
- 可执行
- 不装作确定

示例：

- `这条更像盘前主线预热，先看自主可控和航运替代链是否同步确认。`
- `这条暂时只能当辅助证据，若竞价不扩散，不要把它升级成主线。`

## 6. 成功标准

- 页面里不再只是裸链接
- 每条证据至少能看出标题、来源、摘要、链接
- 每个事件能看到一句判断
- 没模型 key 时仍可正常展示
- 有模型 key 时可升级成模型判断
